# 氢能领域动态调研项目实施计划

## 一、项目概述

### 1.1 任务重述
为氢能领域投资人编制一份**2025年1-12月中国氢能产业动态调研报告**（Word格式），覆盖国家级和31个省级行政区及新疆生产建设兵团的政策动态与产业动态，为投资决策和合作机会评估提供数据支持。

### 1.2 核心约束
- **语言**：全流程中文输出
- **覆盖范围**：国家 + 31省/自治区/直辖市 + 新疆生产建设兵团（共32个地理单元）
- **内容要求**：全文提取，不限制字数；每条动态含标题+正文+URL
- **来源策略**：以公开可访问渠道为主（政府官网、主流财经媒体、行业门户）
- **输出格式**：Word (.docx)，使用docx-js生成，CJK字体支持（Microsoft YaHei）
- **覆盖策略**：全面覆盖32个区域，按实际可获取信息量

### 1.3 已完成的前期调研
已通过3个并行Explore子代理完成全面网络调研，获取了：
- 国家级20+项政策完整清单及核心内容
- 32个地理单元的省级政策与产业动态
- 关键数据：609个项目备案、2712亿投资、574座加氢站、近4万辆燃料电池汽车
- 9个国家级氢能区域试点、41个项目试点名单
- 各省活跃度分级（Tier1/2/3）

## 三、技术方案

### 3.1 技术栈
- **docx-js**（npm包）生成Word文档
- **Node.js** 运行环境
- **sanitize.py** 后处理（去除空白页/TOC问题）
- **Microsoft YaHei** CJK字体

### 3.2 脚本架构

```
/workspace/generate-report/
  package.json
  generate.js              # 主入口
  lib/
    config.js              # 全局配置
    styles.js              # 文档样式
    helpers.js             # 工具函数
    data/
      national-policy.js   # 国家政策数据
      national-industry.js # 国家产业数据
      provinces/
        guangdong.js       # 各省数据（31个文件）
        ...
        xpcc.js            # 新疆生产建设兵团
      appendix.js          # 附录数据
  output/
    中国氢能产业动态研究报告2025.docx
```

### 3.3 关键技术要点
1. **单Section策略**：所有内容放在同一个section的children数组中，避免空白页
2. **章节分页**：使用`pageBreakBefore: true`实现章节间分页
3. **CJK字体**：`font: { ascii: "Microsoft YaHei", eastAsia: "Microsoft YaHei", hAnsi: "Microsoft YaHei" }`
4. **数据与逻辑分离**：每个省份独立数据文件，便于维护和更新
5. **超链接**：使用`ExternalHyperlink`创建可点击URL

## 四、执行步骤

### Step 1：环境准备
- 确认Node.js环境
- 创建项目目录 `/workspace/generate-report/`
- 执行 `npm install docx`
- 确认sanitize.py路径

### Step 2：编写框架代码
- `config.js`：全局常量（标题、日期、字体、颜色）
- `styles.js`：H1-H3标题样式、正文样式
- `helpers.js`：`createEntry()`、`createProvinceChapter()`、`createTable()`、`numToChinese()`

### Step 3：编写国家篇数据
- `national-policy.js`：20+条国家级政策动态（含完整内容摘要和URL）
- `national-industry.js`：15+条国家级产业动态

### Step 4：编写省级数据（按优先级）
- Tier1七省：每省10-20条动态
- Tier2十省：每省5-12条动态
- Tier3十四省：每省2-6条动态
- 新疆生产建设兵团：1-2条

### Step 5：编写附录数据
- 政策文件清单表格
- 加氢站统计表格
- 项目投资汇总表格
- 参考文献列表

### Step 6：编写主生成脚本
- `generate.js`：按顺序加载所有数据模块，构建完整children数组，调用Packer.toBuffer()生成文件

### Step 7：生成与后处理
- 执行 `node generate.js`
- 执行 `python sanitize.py` 后处理
- 验证输出文件

### Step 8：质量检查
- 检查32个地理单元全覆盖
- 检查每条记录含标题+正文+URL
- 检查无空白页
- 检查CJK字体渲染
- 检查超链接可点击

## 六、风险与应对

| 风险 | 应对策略 |
|------|---------|
| 部分省份信息不足 | 明确标注"暂未检索到独立进展"，不编造数据 |
| 脚本文件过大 | 数据与逻辑分离，延迟加载各省模块 |
| 政策URL失效 | 优先使用gov.cn域名，提供文件标题便于搜索 |
| 不同来源数据矛盾 | 以政府官方数据为准，标注差异 |
| CJK字体问题 | Word打开时自动字体替换 |
