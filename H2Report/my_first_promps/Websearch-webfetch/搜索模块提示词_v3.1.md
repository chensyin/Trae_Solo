# 搜索模块提示词 v3.1（独立模块）

> **定位**：本模块是月度调研报告工作流的第一阶段——信息采集，负责从确定关键词到获取所有搜索结果的标题、摘要、全文和链接。
>
> **输入**：报告月份（如 `2026年4月`）
>
> **输出**：结构化的信息采集结果，包含每条信息的标题、摘要、全文关键内容、来源链接、相关度评分

---

## 一、搜索策略总则

### 1.1 搜索控制策略：纯质量驱动

**不设搜索次数上限。** 搜索何时停止，完全由以下质量信号决定：

#### 停止条件（全部满足时停止搜索）

| # | 停止条件 | 说明 |
|---|---------|------|
| 1 | 每个维度至少有 **≥2条 ≥4分**（高质量）结果 | 确保每个维度都有足够权威的信息源 |
| 2 | 最近一轮补充搜索的**新增 ≥3分结果 ≤1条** | 说明该方向已被搜尽，继续搜索收益极低 |
| 3 | 连续 **2轮** 补充搜索均未能提升任何维度的信息覆盖度 | 确认信息采集已饱和 |

#### WebFetch 控制

WebFetch 不设硬性上限，但遵循以下原则：
- 只抓取评分 **≥3分** 且**非重复**的 URL
- 每批最多 **5个并行**
- 抓取失败的 URL 不重复尝试（记录后跳过）

### 1.2 核心原则

1. **信息源头优先**：政策从政府网站获取，产业从行业媒体获取，技术从学术/企业来源获取——不同维度使用不同的搜索策略
2. **定点采集 + 广度搜索结合**：政策维度用 `site:` 定向搜索政府网站，其他维度用通用搜索
3. **去重优先**：每条搜索结果先与已有结果比对，标题/来源/事件重复的只保留信息最全的一条
4. **质量评分**：对每条搜索结果标注 1-5 分，只有 ≥3 分的结果才进入 WebFetch 环节
5. **搜尽为止**：只要还有维度未满足停止条件且补充搜索仍有新增高质量结果，就继续搜索

## 二、搜索执行流程

### 第 1 步：基底词搜索（1次）

```
"氢能 [月份] [年份]"    lr="zh"  num=5
```

作为"温度计"，快速了解当月行业整体动态方向，为后续维度搜索提供选词依据。

---

### 第 2 步：政策维度——定点采集（约 12-18 次搜索）

> **核心原则**：政策信息从发布源头获取，不依赖通用搜索引擎的"碰运气"。
>
> **方法**：使用 `site:` 语法，分层搜索政府网站和权威媒体网站。
>
> **v3.1 优化**：中央部委"先合并后深挖"、解读材料合并搜索、地方政府"动态触发"而非固定全覆盖。

#### 2.1 第一层：中央政府网站（先合并后深挖，约 3-6 次）

**阶段A：合并搜索（1次）——快速扫描哪些部委当月有氢能政策**

```
"氢能" site:gov.cn [月份] [年份]    lr="zh"  num=5
```

> `site:gov.cn` 覆盖国务院及所有部委子域名，一次搜索即可扫描全局。

分析合并搜索结果，识别哪些部委当月有氢能政策动态（通过URL域名判断来源部委）。

**阶段B：对有信号的部委单独深挖（2-5次）**

对阶段A中发现当月有氢能政策动态的部委，执行单独 `site:` 搜索。此外，**国家能源局为默认深挖对象**（无论阶段A是否出现），因为国家能源局是氢能政策最核心的部委，当月有政策的概率较高。

| # | 搜索语句 | 目标部委 | 触发条件 |
|---|---------|---------|---------|
| P0 | `"氢能" site:nea.gov.cn [月份] [年份]` | **国家能源局** | **默认执行（无论阶段A是否出现）** |
| P1 | `"氢能" site:miit.gov.cn [月份] [年份]` | 工信部 | 阶段A结果中出现了 miit.gov.cn |
| P2 | `"氢能" site:ndrc.gov.cn [月份] [年份]` | 发改委 | 阶段A结果中出现了 ndrc.gov.cn |
| P3 | `"氢能" site:mof.gov.cn [月份] [年份]` | 财政部 | 阶段A结果中出现了 mof.gov.cn |
| P4 | `"氢能" site:mee.gov.cn [月份] [年份]` | 生态环境部 | 阶段A结果中出现了 mee.gov.cn |
| P5 | `"氢能" site:most.gov.cn [月份] [年份]` | 科技部 | 阶段A结果中出现了 most.gov.cn |
| P6 | `"氢能" site:mot.gov.cn [月份] [年份]` | 交通运输部 | 阶段A结果中出现了 mot.gov.cn |

> **关键改进**：不再盲目搜索所有8个部委，而是先合并扫描再精准深挖。实测表明通常只有3-4个部委当月有氢能政策，可节省4-5次搜索。

#### 2.2 第二层：政策解读材料（合并搜索，1次）

将三种解读类型合并为一条搜索：

```
"氢能" (解读 OR 答记者问 OR 一图读懂) site:gov.cn [月份] [年份]    lr="zh"  num=5
```

> **关键改进**：v3.0 用3条搜索分别搜解读/答记者问/一图读懂，但三者高度重叠。合并为1条可节省2次搜索。

#### 2.3 第三层：地方政府——动态触发（约 2-5 次）

> **v3.1 核心改进**：不再固定搜索12个省份（实测命中率仅25%），改为"先发现信号，再定向获取"。

**阶段A：从权威媒体发现地方政策信号**

在第2.4层权威媒体搜索完成后，分析媒体搜索结果中提到的**地方政策动态**（如"XX省发布""XX市出台""XX地区试点"等）。

**阶段B：对有信号的地方政府定向搜索**

仅对阶段A中发现有当月政策动态的省份，执行 `site:` 搜索获取政策原文：

```
"氢能" site:[该省发改委或工信厅域名] [月份] [年份]    lr="zh"  num=5
```

**备选方案**：如果权威媒体未发现地方政策信号，但基底词搜索或中央政策中提到了特定省份（如试点申报城市），则对这些省份执行定向搜索。

> **地方政府域名参考**（附录B，按需查阅）：
> 四川 `fgw.sc.gov.cn` / 内蒙古 `fgw.nmg.gov.cn` / 山东 `fgw.shandong.gov.cn` / 甘肃 `fgw.gansu.gov.cn` / 新疆 `fgw.xinjiang.gov.cn` / 广东 `gxt.gd.gov.cn` / 安徽 `jxt.ah.gov.cn` / 河北 `gxt.hebei.gov.cn` / 吉林 `jxt.jl.gov.cn` / 福建 `fjjxw.fujian.gov.cn` / 上海 `gxj.sh.gov.cn` / 湖北 `jxt.hubei.gov.cn`

#### 2.4 第四层：权威新闻媒体（约 3-4 次）

用权威媒体补充政策舆论反应和地方政策动态（同时也是2.3层地方政策信号的来源）：

| # | 搜索语句 | 媒体 |
|---|---------|------|
| M1 | `"氢能" 政策 新华社 [月份] [年份]` | 新华社 |
| M2 | `"氢能" 政策 site:people.com.cn [月份] [年份]` | 人民日报 |
| M3 | `"氢能" 政策 site:ce.cn [月份] [年份]` | 中国经济网/经济日报 |
| M4 | `"氢能" 政策 site:jjckb.cn [月份] [年份]` | 经济参考报 |

> **关键改进**：新华社搜索从 `site:xinhuanet.com` 改为 `"氢能" 政策 新华社`（实测 `site:xinhuanet.com` 无返回）。

#### 2.5 政策维度评分标准（专用）

政策维度的评分标准比通用标准更严格，优先级为：**政府原文 > 政府解读 > 权威媒体报道 > 一般媒体报道**：

| 评分 | 标准 | 来源类型 |
|------|------|---------|
| **5分** | 政策原文全文（含文号、条款、附件），且确认为**当月新发布** | gov.cn / 各部委官网 |
| **5分** | 官方解读材料（答记者问/一图读懂/政策解读），且为当月发布 | gov.cn / 各部委官网 |
| **4分** | 权威媒体对政策的深度报道（含具体数据/条款引用） | 新华社/人民日报/经济日报 |
| **3分** | 一般媒体的政策转载或简讯 | 行业媒体/地方媒体 |
| **2分** | 政策相关但**无法确认为当月新发布**，或内容泛泛 | 自媒体/论坛/存量政策 |
| **1分** | 无关内容 | 广告/爬虫陷阱 |

> **v3.1 新增**：评分标准增加"时效性验证"——如果搜索结果无法确认是当月新发布（如地方政府网站返回的存量政策），降1分处理（从3分降至2分）。

#### 2.6 政策维度"充分"标准

| 检查项 | 充分标准 |
|--------|---------|
| 国家层面政策 | ≥2条（政策原文+解读材料） |
| 地方层面政策 | ≥2条（来自≥1个不同省份，确认为当月新发布） |
| 政策多样性 | 不应全部来自同一份文件 |

> **注意**：如果国家层面政策全部来自同一份文件（如当月只有一份重磅政策），即使数量达标，也应在补充搜索中尝试搜索其他部委是否有配套政策。

### 第 3 步：产业维度——全产业链定点采集（约 15-20 次搜索）

> **核心原则**：产业信息分散在多种渠道中，必须按**信息类型**去不同的权威渠道定点采集。

> **方法**：按"公告系统→专业媒体→财经媒体→展会→全产业链遍历"五层结构搜索。

### 第 5 步：快速评估搜索结果（不抓取，仅评分）

对每条搜索结果执行以下评估（**此步不调用 WebFetch**）：

#### 4.1 通用质量评分标准

| 评分 | 标准 | 操作 |
|------|------|------|
| **5分** | 直接来自政府/企业官网，内容权威，与当月高度相关 | ✅ 必须抓取 |
| **4分** | 来自行业知名媒体，内容详实，包含具体数据/时间/企业名 | ✅ 必须抓取 |
| **3分** | 来自一般媒体/自媒体，内容有一定参考价值 | ⚠️ 视信息缺口决定 |
| **2分** | 内容泛泛，缺乏具体信息，或与当月相关性弱 | ❌ 不抓取 |

#### 4.2 去重规则

- **完全重复**：标题相同或高度相似 → 保留评分最高的
- **来源重复**：同一网站对同一事件的报道 → 保留最早或最详尽的
- **事件重复**：不同媒体对同一事件的报道 → 保留信息量最大的
- **政策重复**：同一政策在不同部委网站发布 → 保留最完整的版本（通常 gov.cn 最全）

#### 6.2 补充搜索策略（循环迭代）
