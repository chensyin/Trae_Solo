# 搜索模块提示词 v4.0（独立模块）

> **定位**：本模块是月度调研报告工作流的第一阶段——信息采集，负责从确定关键词到获取所有搜索结果的标题、摘要、全文和链接。
>
> **输入**：报告月份（如 `2026年4月`）
>
> **输出**：结构化的信息采集结果，包含每条信息的标题、摘要、全文关键内容、来源链接、相关度评分
>
> **v4.0 核心升级**：假设拥有无限资源，核心目标是获得最大化搜索结果。各维度最低30次搜索保障，总计不低于200次。增加跨维度协同机制。产业维度增加交易所/央企招标公告搜索。项目维度强化为"前期+后期"双子维度。

---

## 一、搜索策略总则

### 1.1 搜索控制策略：最大化搜索 + 硬性最低保障

**核心假设：拥有无限搜索资源。核心目标：最大化信息采集，兼顾热点深度和非热点广度。**

#### 1.1.1 硬性最低保障（无论质量信号如何，必须执行）

| 维度 | 最低搜索次数 | 说明 |
|------|-------------|------|
| 政策维度 | **≥30次** | 中央+地方+解读+媒体，全覆盖 |
| 产业维度 | **≥30次** | 交易所+专业媒体+财经媒体+产业链遍历 |
| 技术维度 | **≥30次** | 科研机构+全产业链热点+英文学术+专利 |
| 项目维度 | **≥30次** | 前期（备案/环评/招标）+后期（开工/投产/验收） |
| 海外维度 | **≥30次** | 全球+重点国家+中国企业出海+国际组织+专业媒体 |
| 跨维度协同 | **≥10次** | 热点驱动的跨维度深度搜索 |
| 基底搜索 | **3次** | 多角度基底扫描 |
| **总计** | **≥200次** | |

> **铁律**：每个维度必须执行完最低保障次数后，才能进入"质量评估"环节。最低保障是**底线**，不是**上限**。

#### 1.1.2 最大化搜索原则

在完成最低保障后，继续执行补充搜索，直到以下**全部条件**满足：

| # | 停止条件 | 说明 |
|---|---------|------|
| 1 | 每个维度至少有 **≥5条 ≥4分**（高质量）结果 | 确保每个维度信息充足 |
| 2 | 最近一轮补充搜索的**新增 ≥3分结果 ≤1条** | 说明该方向已被搜尽 |
| 3 | 连续 **3轮** 补充搜索均未能提升任何维度的信息覆盖度 | 确认信息采集已饱和 |

#### 1.1.3 热点与非热点兼顾策略

| 搜索类型 | 占比 | 说明 |
|---------|------|------|
| 热点深度搜索 | ~40% | 围绕基底搜索识别的当月热点，多角度、多来源深入挖掘 |
| 非热点广度搜索 | ~40% | 按维度策略模板系统性遍历，确保不遗漏任何细分领域 |
| 跨维度协同搜索 | ~10% | 围绕热点在多个维度间交叉搜索 |
| 补充搜索 | ~10% | 基于覆盖度评估的查漏补缺 |

#### 1.1.4 WebFetch 控制

- 只抓取评分 **≥3分** 且**非重复**的 URL
- 每批最多 **5个并行**
- 抓取失败的 URL 不重复尝试（记录后跳过）
- **不设 WebFetch 总数上限**

### 1.2 核心原则

1. **信息源头优先**：政策从政府网站获取，产业从交易所公告和行业媒体获取，技术从学术/企业来源获取，项目从审批/招标平台获取——不同维度使用不同的搜索策略
2. **定点采集 + 广度搜索结合**：每个维度先用 `site:` 定点采集权威来源，再用通用搜索补充广度
3. **去重优先**：每条搜索结果先与已有结果比对，标题/来源/事件重复的只保留信息最全的一条
4. **质量评分**：对每条搜索结果标注 1-5 分，只有 ≥3 分的结果才进入 WebFetch 环节
5. **搜尽为止**：先完成最低保障搜索，再根据质量信号决定是否继续补充搜索
6. **热点驱动协同**：基底搜索识别的当月热点必须触发每个维度的定向深度搜索

## 二、搜索执行流程

### 第 1 步：多角度基底搜索（3次）

> **v4.0 改进**：从1次扩展为3次，多角度扫描当月热点，确保不遗漏。

| # | 搜索语句 | 扫描角度 |
|---|---------|---------|
| B1 | `"氢能" [月份] [年份]` lr="zh" num=5 | 行业综合动态 |
| B2 | `"氢能" (政策 OR 项目 OR 突破 OR 签约) [月份] [年份]` lr="zh" num=5 | 重大事件 |
| B3 | `hydrogen energy China [月份英文] [年份]` lr="en" num=5 | 国际视角下的中国氢能 |

**基底分析输出**（在执行各维度搜索前必须完成）：

```
=== 当月热点识别 ===
热点1：[事件名称] —— 相关维度：[政策/产业/技术/项目/海外]
热点2：[事件名称] —— 相关维度：[...]
热点3：[事件名称] —— 相关维度：[...]
非热点但需覆盖的常规方向：[...]
```

> **关键**：识别出的每个热点，必须在后续每个相关维度中触发至少1次定向搜索（见"跨维度协同机制"）。

#### 2.7 政策维度评分标准（专用）

| 评分 | 标准 | 来源类型 |
|------|------|---------|
| **5分** | 政策原文全文（含文号、条款、附件），且确认为**当月新发布** | gov.cn / 各部委官网 |
| **5分** | 官方解读材料（答记者问/一图读懂/政策解读），且为当月发布 | gov.cn / 各部委官网 |
| **4分** | 权威媒体对政策的深度报道（含具体数据/条款引用） | 新华社/人民日报/经济日报 |
| **3分** | 一般媒体的政策转载或简讯 | 行业媒体/地方媒体 |
| **2分** | 政策相关但**无法确认为当月新发布**，或内容泛泛 | 自媒体/论坛/存量政策 |
| **1分** | 无关内容 | 广告/爬虫陷阱 |

> **时效性验证**：如果搜索结果无法确认是当月新发布（如地方政府网站返回的存量政策），降1分处理。

#### 2.8 政策维度"充分"标准

| 检查项 | 充分标准 |
|--------|---------|
| 国家层面政策 | ≥3条（来自≥2个不同部委） |
| 地方层面政策 | ≥5条（来自≥3个不同省份，确认为当月新发布） |
| 政策解读 | ≥2条（官方解读+媒体/专家解读） |
| 政策多样性 | 不应全部来自同一份文件 |
| 政策影响分析 | ≥1条（政策影响/行业反应） |

| **3分** | 来自一般媒体/自媒体，内容有一定参考价值 | ⚠️ 视信息缺口决定 |
| **2分** | 内容泛泛，缺乏具体信息，或与当月相关性弱 | ❌ 不抓取 |
| **1分** | 广告/重复/无关内容 | ❌ 不抓取 |

#### 8.2 去重规则

- **完全重复**：标题相同或高度相似 → 保留评分最高的
- **来源重复**：同一网站对同一事件的报道 → 保留最早或最详尽的
- **事件重复**：不同媒体对同一事件的报道 → 保留信息量最大的
- **政策重复**：同一政策在不同部委网站发布 → 保留最完整的版本（通常 gov.cn 最全）
- **跨维度重复**：同一条信息可能同时属于产业和项目维度 → 按主要属性归入一个维度，在另一维度中标注"详见XX维度"
