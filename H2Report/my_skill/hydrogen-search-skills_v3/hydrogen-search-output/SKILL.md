# 氢能搜索质量评估与输出

## 概述

本Skill负责：
1. 协同搜索触发（基于热点）
2. 信息质量评估与去重
3. 覆盖度评估与补充搜索
4. WebFetch并行抓取全文
5. JSON格式最终输出

---

## 第一部分：协同搜索触发

### 1.1 协同搜索规则

当基底搜索识别出当月热点后，按以下规则触发协同搜索：

```
对每个识别出的热点：
  1. 政策维度：搜索该热点的政策背景/政策支持/政策影响（1次）
  2. 产业维度：搜索该热点相关的企业动态/市场反应/产业链影响（1次）
  3. 技术维度：搜索该热点涉及的技术路线/技术突破（1次）
  4. 项目维度：搜索该热点相关的项目进展/项目签约（1次）
  5. 海外维度：搜索该热点的国际对比/海外类似案例（1次）
```

### 1.2 协同搜索语句模板

假设热点为 **"氢能综合应用试点政策发布"**：

| # | 维度 | 搜索语句 | 目的 |
|---|------|---------|------|
| C1 | 政策 | `"氢能综合应用试点" (配套 OR 细则 OR 申报指南) [月份] [年份]` | 搜索配套政策 |
| C2 | 产业 | `"氢能综合应用试点" (企业反应 OR 机会 OR 布局) [月份] [年份]` | 搜索产业反应 |
| C3 | 技术 | `"氢能综合应用试点" (技术路线 OR 技术要求 OR 指标) [月份] [年份]` | 搜索技术关联 |
| C4 | 项目 | `"氢能综合应用试点" (申报 OR 项目储备 OR 候选) [月份] [年份]` | 搜索项目申报 |
| C5 | 海外 | `hydrogen pilot program subsidy (comparison OR benchmark) [年份]` lr="en" | 国际对比 |

假设热点为 **"CIHC中国氢能展"**：

| # | 维度 | 搜索语句 | 目的 |
|---|------|---------|------|
| C6 | 政策 | `"氢能展" OR "CIHC" (政策发布 OR 规划发布) [月份] [年份]` | 展会上的政策发布 |
| C7 | 产业 | `"CIHC" OR "氢能展" (签约 OR 订单 OR 合作 OR 投资) [月份] [年份]` | 展会签约/订单 |
| C8 | 技术 | `"CIHC" OR "氢能展" (新品 OR 首发 OR 技术 OR 创新) [月份] [年份]` | 展会技术发布 |
| C9 | 项目 | `"CIHC" OR "氢能展" (项目 OR 落地 OR 开工) [月份] [年份]` | 展会项目落地 |
| C10 | 海外 | `"CIHC" OR "中国氢能展" (国际 OR 海外企业 OR foreign) [月份] [年份]` | 国际参展企业 |

### 1.3 通用协同搜索模板

对于任意识别出的热点，按以下模板构造协同搜索：

| 维度 | 搜索语句模板 | 目的 |

## 第二部分：信息质量评估与去重

### 2.1 通用质量评分标准

| 评分 | 标准 | 操作 |
|------|------|------|
| **5分** | 直接来自政府/企业官网，内容权威，与当月高度相关 | ✅ 必须抓取 |
| **4分** | 来自行业知名媒体，内容详实，包含具体数据/时间/企业名 | ✅ 必须抓取 |
| **3分** | 来自一般媒体/自媒体，内容有一定参考价值 | ⚠️ 视信息缺口决定 |
| **2分** | 内容泛泛，缺乏具体信息，或与当月相关性弱 | ❌ 不抓取 |
| **1分** | 广告/重复/无关内容 | ❌ 不抓取 |

### 2.2 去重规则

| 重复类型 | 处理方式 |
|---------|---------|
| **完全重复**：标题相同或高度相似 | 保留评分最高的 |
| **来源重复**：同一网站对同一事件的报道 | 保留最早或最详尽的 |
| **事件重复**：不同媒体对同一事件的报道 | 保留信息量最大的 |
| **政策重复**：同一政策在不同部委网站发布 | 保留最完整的版本（通常gov.cn最全） |
| **跨维度重复**：同一条信息可能同时属于产业和项目维度 | 按主要属性归入一个维度，在另一维度中标注"详见XX维度" |

### 2.3 去重标记字段

```json
{
  "is_duplicate": true,
  "duplicate_of": "P0-1"
}
```

## 第三部分：覆盖度评估与补充搜索

### 3.1 覆盖度评估表

| 维度 | ≥3分结果数 | 是否充分？ | 判断标准 |
|------|-----------|-----------|---------|
| 政策 | _ 条 | ☐充分 ☐不足 | ≥3条国家+≥5条地方 |
| 产业 | _ 条 | ☐充分 ☐不足 | ≥10条企业动态+≥3条交易所公告 |
| 技术 | _ 条 | ☐充分 ☐不足 | ≥8条国内研发+≥3条英文学术 |
| 项目 | _ 条 | ☐充分 ☐不足 | ≥5条前期+≥3条后期 |
| 海外 | _ 条 | ☐充分 ☐不足 | ≥3条全球+≥2条出海 |

### 3.2 补充搜索策略

**仅针对"不足"的维度**，已"充分"的维度不再搜索。

```
┌─────────────────────────────────────┐
│  检查停止条件                        │
│  ├─ 全部满足 → 跳到WebFetch阶段      │
│  └─ 未满足 ↓                        │
│  识别最需要补充的维度                  │
│  构造针对性补充搜索（1-3条）           │
│  执行搜索 → 评分 → 去重 → 更新覆盖度   │
│  └── 回到循环顶部 ──┘                │
└─────────────────────────────────────┘
```

### 3.3 补充搜索语句模板

| 缺口维度 | 补充搜索策略 |
|---------|-------------|
| 产业不足 | `"电解槽/燃料电池/储氢 [中标/订单/出口] [月份] [年份]"` 或 `"[具体企业名] 氢能 [融资/中标/签约] [年份]"` |
| 技术不足 | `"[具体技术路线] [研发机构] 突破/进展 [月份] [年份]"` 或 `"hydrogen [technology] breakthrough [月份英文] [年份]"` lr="en" |
| 项目不足 | `"氢能 [招标/备案/环评] [月份] [年份]"` 或 `"氢能 [项目类型] [具体省份/城市] [月份] [年份]"` |
| 海外不足 | `"hydrogen [国家名] [项目/企业名] [月份英文] [年份]"` lr="en" |
| 政策不足（地方） | `"氢能" site:[未搜索省份发改委域名] [月份] [年份]` |

### 3.4 换词原则

- 更换同义词（如"融资"→"投资"→"战略投资"）
- 更换具体主体（如"内蒙古"→"新疆"→"甘肃"）
- 更换搜索角度（如"政策名称"→"政策解读"→"政策影响"）
- 如果某维度连续3轮补充搜索均无新增≥3分结果，标记为"已搜尽"

## 第四部分：WebFetch并行抓取

### 4.1 分批策略

- 每批最多5个并行
- 按维度分组，同一维度的URL放在同一批
- 优先抓取评分高的URL

### 4.2 抓取后信息提取格式

```json
{
  "id": "P0-1",
  "search_id": "P0",
  "title": "页面标题（从页面中提取，不得编造）",
  "source": "网站/媒体名称",
  "url": "https://xxx.xxx/xxx（搜索阶段实际返回的完整URL）",
  "publish_date": "2026-03-16（具体日期，如无则填null）",
  "dimension": "policy",
  "sub_category": "national_policy",
  "score": 5,
  "key_info": {
    "issuer": "工信部/财政部/发改委",
    "doc_number": "工信部联节〔2026〕59号",
    "core_content": "以揭榜挂帅方式遴选5个城市群",
    "details": ["燃料电池汽车", "绿色氨醇", "氢基化工原料替代"]
  },
  "full_text_snippet": "正文关键段落摘录（200-500字）",
  "quality": "high",
  "notes": null
}
```

### 4.3 WebFetch后二次验证

- 如果抓取内容与标题严重不符，降分处理
- 如果抓取内容为快讯（<100字），`quality`标记为`"low"`
- 如果URL指向首页/列表页而非具体文章，`score`降为1分，`url`标记为`null`

## 第六部分：URL完整性铁律

### 6.1 禁止的URL格式

```
❌ 企业首页：https://www.sinopec.com.cn/
❌ 政府首页：https://www.ndrc.gov.cn/
❌ 栏目列表页：https://www.h2-view.com/category/europe
❌ 搜索结果页：https://h2.china-nengyuan.com/news/?keyword=制氢
❌ 带参数的跳转页：https://cj.sina.cn/article/norm_detail?url=...
❌ 域名异常：http://2fwww.tanjiaoyi.com/
```

### 6.2 正确的URL格式

```
✅ https://m.mof.gov.cn/czxw/202603/t20260316_3985379.htm
✅ https://disc.static.szse.cn/download/disc/disk03/finalpage/2026-03-07/86ba3272.PDF
✅ https://www.miit.gov.cn/jgsj/jns/gzdt/art/2026/art_xxx.html
```

### 6.3 来源偏好规则

```
⚠️ 政策维度（dimension=policy）：
  第1优先：政府部门官网（gov.cn/miit.gov.cn/ndrc.gov.cn等）
  第2优先：权威中文媒体（xinhuanet.com/people.com.cn/ce.cn）
  第3优先：行业专业媒体
  禁止：英文媒体URL、企业官网URL

⚠️ 非政策维度（industry/technology/project/overseas）：
  第1优先：中文来源（中文媒体/政府网站/交易所公告）
  第2优先：中文行业媒体
  第3优先：英文来源（仅在中文来源确实不存在时使用）
  禁止：企业官网首页

⚠️ 海外维度（dimension=overseas）：
  第1优先：中国企业的中文报道
  第2优先：国际权威媒体（reuters.com/ft.com/bloomberg.com）
  第3优先：海外专业媒体
  禁止：企业官网首页
```

## 第七部分：search_trace溯源机制

### 7.1 溯源字段说明

`search_trace` 字段记录发现该信息所经过的搜索语句编号：

```
信息：三部门发布氢能综合应用试点通知
search_trace: ["P0", "P1", "P2", "P10", "P32"]
含义：该信息在P0、P1、P2、P10、P32搜索中均被返回
用途：
  ① 验证信息的可靠性（被多个搜索交叉验证）
  ② 追溯信息发现路径
  ③ 评估搜索策略的有效性
```

### 7.2 溯源字段格式

```json
{
  "search_trace": ["P0", "P1", "P2"]
}
```

## 第八部分：importance评级标准

| 等级 | 标准 | 示例 |
|------|------|------|
| `critical` | 当月最重要的1-3个事件，对行业产生重大影响 | 三部门试点政策、CIHC展会、万吨级项目投产 |
| `high` | 对特定领域有重要影响 | 央企招标、重大签约、技术突破 |
| `medium` | 有一定参考价值 | 一般企业动态、地方政策、常规项目 |
| `low` | 信息不充分或影响较小 | 快讯、旧闻、内容泛泛 |

**限制**：`critical` 条目 ≤5 条
